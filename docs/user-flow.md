# User Flow — AI 역량검사 전략게임 연습 서비스

> 출처: `docs/기획서_김지민.pdf`. UI/UX 작업을 위해 기획서의 User Flow 섹션을 화면·상태 단위로 구체화했다.

## 한 줄 요약

사용자는 **학습 모드**에서 게임 규칙과 문제 유형을 이해하고, **실전 분량 모드**에서 제한시간 아래 연속 문제를 풀며 실제 검사에 대비한다. 게임 종료 후에는 총점이 아닌 **문항별 정답·오답**을 확인하며 자신의 판단을 검토한다.

## 전체 플로우

```
홈
 │  서비스 소개, 전략게임 목록
 ▼
게임 선택
 │  도형 돌리기 / 마법약 만들기 / 도형 순서 기억하기 / 음식·버스 번호 기억하기
 ▼
모드 선택
 │
 ├─▶ 학습 모드 ──┐
 │               │
 └─▶ 실전 분량 모드 ┤
                 ▼
              결과 화면 (문항별 정답·오답 확인)
                 │
                 ├─▶ 유사 문제 재도전
                 ├─▶ 같은 게임 재도전
                 └─▶ 다른 게임 선택 (게임 선택으로 복귀)
```

## 화면별 상세 플로우

### 1. 홈

- 서비스 소개
- 전략게임 목록 노출 (도형 돌리기 / 마법약 만들기 / 도형 순서 기억하기 / 음식·버스 번호 기억하기)
- 진입점: 게임 카드 클릭 → **게임 선택**

### 2. 게임 선택

- 4개 게임 중 하나 선택
- 진입점: 게임 확정 → **모드 선택**

### 3. 모드 선택

- 옵션: **학습 모드** | **실전 분량 모드**
- **첫 진입 안내(soft-gate)**: 해당 게임의 학습 모드 완료 이력이 없는 상태로 실전 분량 모드를 누르면, 강제 이동시키지 않고 확인 모달을 띄운다 — "이 게임은 아직 학습 전이에요. 그래도 실전 모드로 시작할까요?" (계속하기 / 학습 모드로 이동). 선택권은 유지하되 기본 추천만 유도한다.

### 4. 학습 모드

1. 게임 규칙과 조작 방법 확인
2. **문제 개수를 사용자가 직접 입력해 선택** (실전 분량 모드와의 차이 — 실전 모드는 실제 검사와 유사한 고정 분량, 학습 모드는 사용자가 원하는 만큼 연습)
3. 예제 문제와 단계별 문제 풀이
4. **문항 풀이 직후 즉시 정답·오답 피드백** (실전 모드와의 차이점 — 학습 모드는 문제를 풀 때마다 바로 확인해야 판단을 그 자리에서 교정할 수 있다)
5. (도형 돌리기 게임인 경우) 필요 시 **도형 회전 시뮬레이터** 사용
6. 분기: 유사 문제(새 문제) 재도전 **또는** 실전 분량 모드로 이동

### 5. 실전 분량 모드

1. 문제 수와 제한시간 적용 (실제 검사와 유사한 분량, **전체 세트 기준 제한시간** — 문항별로 시간을 끊지 않는다. 실제 검사의 "전체 시간 안배 판단"까지 연습 대상에 포함하기 위함)
2. 여러 문제를 연속으로 풀이. **정답·오답은 세트를 모두 마칠 때까지 노출하지 않음** (학습 모드와의 차이 — 중간에 결과를 알면 시간 압박 감각이 흐트러짐)
3. 시간 종료 또는 사용자 제출
4. 전체 결과와 문항별 정답 확인 (**결과 화면**으로 이동)
5. 분기: 같은 게임 재도전(**새 문제 세트** — 정답을 이미 확인했으므로 동일 문제 재사용 시 학습이 아닌 암기가 됨) **또는** 다른 게임 선택 (게임 선택으로 복귀)

### 5-1. 도형 회전 시뮬레이터 (보조 도구, 학습 모드 내 진입)

- 4×4 격자에서 사용자가 칸을 선택하면 해당 칸이 색으로 채워져 도형을 이룬다
- 조작 버튼: **왼쪽 45도 회전** / **오른쪽 45도 회전** / **좌우 반전** / **위아래 반전**
- 버튼을 누를 때마다 격자 위 도형이 실제로 변형되는 과정을 시각적으로 확인
- 도형 돌리기 게임 규칙 학습을 위한 독립 보조 화면 (모드 선택과 무관하게 학습 모드에서 열람 가능)

### 6. 결과 화면 (Result Dashboard)

- 종합 정확도(Accuracy) 표시
- **문항별 정답·오답** 리스트 — 총점만이 아니라 각 문항의 판단을 검토
- CTA: 문항별 결과 보기 / 다시 풀기
- 분기:
  - 유사 문제 재도전 (학습 모드에서)
  - 같은 게임 재도전
  - 다른 게임 선택 → 게임 선택 화면으로 복귀

## 게임별 플로우 차이

공통 플로우(홈 → 게임 선택 → 모드 선택 → 학습/실전 → 결과)는 4개 게임이 공유하지만, 학습 모드 내부 콘텐츠와 보조 도구는 게임마다 다르다.

| 게임 | 판단 유형 | 전용 보조 도구 |
|---|---|---|
| 도형 돌리기 | Before→After 회전·반전 경로 판단 | 도형 회전 시뮬레이터 (4×4 격자, 좌/우 45도 회전 + 좌우·상하 반전) |
| 마법약 만들기 | 미공개 조합 규칙 추론 (중간에 규칙 변경) | 없음 |
| 도형 순서 기억하기 | N-back 방식 순서 기억 | 없음 |
| 음식·버스 번호 기억하기 | 다자 정보 암기 후 공통 항목 회상 | 없음 |

각 게임 규칙의 상세 설명은 아래 "게임 규칙" 섹션 참고.

도형 회전 시뮬레이터는 **도형 돌리기 게임의 학습 모드에서만** 노출된다. 공통 플로우 다이어그램의 "필요 시" 표시는 이 조건부 노출을 뜻한다.

## 게임 규칙

> 잡다(JOBDA) 실제 서비스의 동일 계열 게임과 응시 후기를 참고해 정리했다. 문항 데이터가 아닌 게임 메커니즘 참고용이다.
> 실제 화면 디자인 레퍼런스는 `design/` 폴더 참고 (도형회전하기, 도형순서기억하기, 음식, 버스 각 2장씩).

### 도형 돌리기

- 화면 왼쪽에 Before 도형과 After 도형(목표)이 함께 제시된다.
- 화면 오른쪽 상단에 조작 버튼 4개: **좌 45도 회전 / 우 45도 회전 / 좌우 반전 / 위아래 반전**.
- 사용자는 Before → After가 되는 조작 순서를 이 버튼을 눌러 직접 재현한다 (객관식이 아니라 조작 시퀀스를 직접 구성하는 방식).
- 버튼을 누를 때마다 화면 오른쪽 하단 **2×4칸(총 8칸) 이력 박스**에 누른 순서대로 화살표 아이콘이 기록된다.
- 이력 관리: **전체 초기화** 버튼, **마지막 1개 취소(되돌리기)** 버튼 제공.
- 시퀀스를 다 구성한 뒤 **"제출" 버튼**을 눌러 채점한다 (버튼을 누를 때마다 Before 도형이 실시간으로 바뀌는 미리보기 방식이 아님).
- 학습 모드에서는 도형 회전 시뮬레이터로 이 조작 버튼들을 먼저 연습한 뒤 문제를 푼다 (동일한 조작 버튼 세트를 공유).
- **2라운드 구성**, 라운드마다 도형 종류가 다르다.
  - 1라운드: `J·R·Q·L·G` 문자형 도형 세트를 회전·반전
  - 2라운드: 4×4 정사각형 격자(16칸) 중 특정 칸이 색칠된 도형을 회전·반전 (도형 회전 시뮬레이터와 같은 포맷)

### 마법약 만들기

- 재료를 조합하면 결과물(빨간약/파란약 등)이 나오는데, 조합 규칙은 사전에 공개되지 않는다.
- 사용자는 여러 번 조합을 시도하며 규칙을 추론해야 하고, 규칙은 게임 중간에 바뀔 수 있다.
- 확률·추론 요소가 커서 체감 난이도가 가장 높은 유형 중 하나. 학습 모드에서는 "규칙은 언제든 바뀔 수 있다"는 점을 먼저 안내한다.

#### 콘텐츠 정의

> 보류 — 나머지 3개 게임 콘텐츠 정의를 먼저 마무리한 뒤 다시 다룬다.

### 도형 순서 기억하기

- 도형이 순서대로 하나씩 제시된다 (N-back 방식).
- 1라운드: 현재 도형이 **2개 전** 도형과 같은지/다른지 판단
- 2라운드: 현재 도형이 **2개 전 또는 3개 전** 도형과 같은지, 아니면 둘 다 다른지 판단
- 도형에 이름(동그라미·마름모·별 등)을 붙이면 기억하기 쉬워진다는 실제 사용자 팁을 학습 모드 안내 문구로 제공한다.
- **시작 워밍업**: 문제 풀이 전 도형 3개를 순서대로 각 3초씩 먼저 보여준다 (2개 전/3개 전 비교가 가능하도록 기준 도형을 먼저 쌓는 구간).
- **화면 구성**: 왼쪽에 현재 제시되는 도형, 화면 상단 중앙에 판단 지시문(예: "화면에 제시되는 도형이 두 번째 전 도형과 같은지 판단해주세요").
- **키보드 입력 3종**: `Space bar` / `←` (두 번째 전과 같음) / `→` (세 번째 전과 같음). `Space bar`는 "둘 다 다름"으로 추정 — 확인 필요.

#### 콘텐츠 정의 (초안 — 확인 필요)

| 항목 | 값 |
|---|---|
| 도형 풀 | 원 · 세모 · 네모 (3종, 일단 이 범위로 시작) |
| 라운드 구성 | 1라운드: 현재 도형이 2개 전과 같은지/다른지 · 2라운드: 2개 전 또는 3개 전과 같은지, 모두 다른지 |
| 워밍업 | 도형 3개를 순서대로 각 3초씩 먼저 노출 |
| 입력 방식 | 키보드 `Space` / `←`(2번째 전과 같음) / `→`(3번째 전과 같음) |
| 실전 모드 문항 수 | 총 30개 |
| 노출 시간(문항당) | 미정 |

### 음식·버스 번호 기억하기

순간 기억 게임이지만, **음식 파트와 버스 번호 파트는 서로 다른 규칙**을 쓴다 — 음식은 "반복 등장한 것 찾기", 버스 번호는 "등장하지 않은 것 찾기"로 정답 판단 방향이 반대다.

#### 진행 방식 — 음식 파트

각 턴은 **사람 이름 + 그 사람이 고른 음식**으로 구성된다 (예: "철수" 이름이 화면 중앙 상단에 뜨고, 그 아래 나무 테이블 배경 위에 동그란 검정 그릇 음식들이 나열됨).

**3개 버전**
1. 1턴: "OO" 이름 + 음식 3개를 보여줌 (1초)
2. 2턴: "OO" 이름 + 음식 3개를 보여줌 (1초)
3. 3턴: "OO" 이름 + 음식 3개를 보여줌 (1초)
4. 최종 화면("어떤 메뉴를 선택하면 좋을까요?"): 흰 배경에 음식 6개(2행×3열)를 제시. 1~3턴에서 보여준 음식들 중 **겹치는(반복 등장한) 음식**을 3초 안에 고른다.
5. 겹치는 음식은 항상 정확히 1개가 되도록 출제한다 (정답이 복수이거나 없는 경우는 없음).

**4개 버전**
- 위와 동일한 구조, 턴마다 보여주는 음식 개수만 3개 → 4개로 늘어남 (난이도 상승).

#### 진행 방식 — 버스 번호 파트

각 턴도 음식과 동일하게 **사람 이름 + 그 사람이 탄 버스 번호**로 구성된다 (이름이 중앙 상단, 아래 도로/버스 정류장 배경 위에 버스 번호 말풍선이 나열됨).

1. 1턴: "OO" 이름 + 버스 번호를 1개 또는 2개 무작위로 보여줌
2. 2턴: "OO" 이름 + 버스 번호를 1개 또는 2개 무작위로 보여줌
3. 3턴: "OO" 이름 + 버스 번호를 1개 또는 2개 무작위로 보여줌
4. 최종 화면("누구도 탑승하지 않은 버스 번호는 몇 번일까요?"): 5개 선택지(가로 1열, 색상 박스)를 제시. 1~3턴에서 **한 번도 나오지 않은** 버스 번호를 고른다.

**번호 생성 규칙**
- 한 문항 안에서 등장하는 모든 버스 번호(1~3턴 노출분 + 최종 5개 선택지)는 서로 겹치지 않는다 (중복 없음).
- 한 문항 안에 **1자리수, 2자리수, 3자리수 번호가 각각 최소 1개 이상** 포함되어야 한다.

턴당 노출 1초, 최종 응답 제한 3초 (음식 파트와 동일).

| 항목 | 값 |

## 비로그인 진행 기록

MVP는 "복잡한 회원 기능"을 초기 제외 범위로 두지만(→ MVP 범위 참고), 검증 지표에는 게임별 재도전 횟수·정답 확인 사용률처럼 진행 기록이 필요한 지표가 포함되어 있다. 이를 로그인 없이 만족시키기 위한 저장 방식:

- 브라우저 로컬 저장소(localStorage 또는 IndexedDB)에 디바이스 기준으로 게임별 학습 완료 여부, 시도 횟수, 최근 정확도를 저장한다.
- 계정이 없으므로 기기를 바꾸면 기록이 승계되지 않는다 — 이는 MVP 단계에서 허용 가능한 제약으로 명시한다.
- 지표 집계(재도전 횟수, 완료율 등)는 개인 식별 정보 없이 익명 세션 단위 이벤트로 전송한다 (`skills/frontend/SKILL.md`의 PII 비노출 규칙과 일치).

## MVP 범위

1. 도형 돌리기 게임 (회전·반전 판단)
2. 마법약 만들기 게임 (규칙 추론과 조합 판단)
3. 도형 순서 기억하기 (순서 기억과 집중력)
4. 음식·버스 번호 기억하기 (정보 기억과 회상)
5. 문항별 정답·오답 확인 (게임 종료 후 판단 검토)
6. 도형 회전 시뮬레이터 (직접 조작하며 원리 이해)
7. 실전 분량 연습 모드 (문제 수 + 제한시간 적용 연속 풀이)
